<objective>
Enable PostgreSQL Row-Level Security (RLS) on all user-owned tables and define policies that restrict each row to the user identified by the session variable `app.current_user_id`.

Purpose: Database-level enforcement that even if application code contains a bug, cross-user data leakage is impossible because PostgreSQL itself filters rows.
Output: RLS enabled on 13 tables with idempotent policies; users table left policy-free for auth lookups.
</objective>

Add a new private async createRlsPolicies() method to DbService and call it from onModuleInit() after createTables().

The method must run the following SQL in a single client.query() call (idempotent — uses DROP POLICY IF EXISTS + CREATE POLICY pattern for each table):

TABLES WITH DIRECT user_id COLUMN (accounts, contacts, memory_banks, api_keys, refresh_tokens, password_resets):
For each table, the policy expression is:
user_id = current_setting('app.current_user_id', true)

TABLES WITH account_id → accounts.user_id (jobs, raw_events, memories):
For each table, the policy expression uses an EXISTS subquery:
EXISTS (SELECT 1 FROM accounts a WHERE a.id = {table}.account_id AND a.user_id = current_setting('app.current_user_id', true))

TABLES VIA memories (memory_links, memory_contacts):
memory_links: EXISTS (SELECT 1 FROM memories m WHERE m.id = memory_links.src_memory_id AND EXISTS (SELECT 1 FROM accounts a WHERE a.id = m.account_id AND a.user_id = current_setting('app.current_user_id', true)))
memory_contacts: EXISTS (SELECT 1 FROM memories m WHERE m.id = memory_contacts.memory_id AND EXISTS (SELECT 1 FROM accounts a WHERE a.id = m.account_id AND a.user_id = current_setting('app.current_user_id', true)))

TABLES VIA contacts (contact_identifiers, merge_dismissals):
contact_identifiers: EXISTS (SELECT 1 FROM contacts c WHERE c.id = contact_identifiers.contact_id AND c.user_id = current_setting('app.current_user_id', true))
merge_dismissals: EXISTS (SELECT 1 FROM contacts c WHERE c.id = merge_dismissals.contact_id_1 AND c.user_id = current_setting('app.current_user_id', true))

For each of the 13 tables, the DDL block pattern is:

```sql
ALTER TABLE {table} ENABLE ROW LEVEL SECURITY;
ALTER TABLE {table} FORCE ROW LEVEL SECURITY;
DROP POLICY IF EXISTS rls_{table}_select ON {table};
DROP POLICY IF EXISTS rls_{table}_insert ON {table};
DROP POLICY IF EXISTS rls_{table}_update ON {table};
DROP POLICY IF EXISTS rls_{table}_delete ON {table};
CREATE POLICY rls_{table}_select ON {table} FOR SELECT USING ({expression});
CREATE POLICY rls_{table}_insert ON {table} FOR INSERT WITH CHECK ({expression});
CREATE POLICY rls_{table}_update ON {table} FOR UPDATE USING ({expression});
CREATE POLICY rls_{table}_delete ON {table} FOR DELETE USING ({expression});
```

FORCE ROW LEVEL SECURITY ensures table owners (superusers aside) also obey RLS.

The current_setting second arg `true` means "return NULL instead of ERROR if the setting is unset" — this prevents errors during table creation before any session var is set.

Call createRlsPolicies() in onModuleInit() after createTables() and before the success log:

```typescript
await this.createTables();
await this.dropLegacyTables();
await this.createRlsPolicies();
this.logger.log('PostgreSQL connected and tables ensured');
```

Do NOT add RLS to: users, settings, connector_credentials.
