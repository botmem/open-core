<task type="auto">
  <name>Task 2: Rewrite DbService to pg Pool</name>
  <files>apps/api/src/db/db.service.ts</files>
  <action>
Complete rewrite of DbService:

1. **Imports:** Replace `better-sqlite3` and `drizzle-orm/better-sqlite3` with `pg` (Pool) and `drizzle-orm/node-postgres` (drizzle, NodePgDatabase).

2. **Properties:**
   - Remove `public sqlite!: Database.Database`
   - Change `public db!: BetterSQLite3Database<typeof schema>` to `public db!: NodePgDatabase<typeof schema>`
   - Add `private pool!: Pool`

3. **onModuleInit:** Make async.
   - Create Pool: `new Pool({ connectionString: this.config.databaseUrl, max: 20, idleTimeoutMillis: 30000, connectionTimeoutMillis: 5000 })`
   - Init Drizzle: `this.db = drizzle(this.pool, { schema })`
   - Call `await this.createTables()`
   - Log: `PostgreSQL connected and tables ensured`

4. **onModuleDestroy:** Add `async onModuleDestroy() { await this.pool.end(); }`

5. **healthCheck():** Public async method: acquire client, `SELECT 1`, release, return boolean. This replaces the old `sqlite.prepare('SELECT 1').get()` pattern used by health.controller.ts.

6. **createTables():** Rewrite all SQL to PostgreSQL syntax:
   - `CREATE EXTENSION IF NOT EXISTS pg_trgm;`
   - All tables use Postgres types: `BOOLEAN DEFAULT false` instead of `INTEGER DEFAULT 0`, `TIMESTAMPTZ` instead of `TEXT` for timestamps, `JSONB` for factuality/weights/avatars/contacts.metadata, `DOUBLE PRECISION` for real columns
   - Remove FTS5 virtual table, triggers -- FTS is handled by GIN indexes on tsvector
   - Add GIN indexes: `CREATE INDEX IF NOT EXISTS idx_memories_fts ON memories USING gin (to_tsvector('english', text) || to_tsvector('simple', text));`
   - Add trigram index: `CREATE INDEX IF NOT EXISTS idx_memories_trgm ON memories USING gin (text gin_trgm_ops);`
   - Keep all regular indexes (contact_identifiers, memory_contacts, etc.)
   - Use `CREATE TABLE IF NOT EXISTS` for all tables (same pattern)

7. **Remove SQLite-only migrations:** Delete all the `ALTER TABLE ... ADD COLUMN` try/catch blocks -- these were SQLite schema evolution hacks. Postgres tables are created with the full schema from the start (no need to add columns after the fact). Also remove `migrateUserOwnership()`, `migrateEncryption()`, `migrateMemoryEncryption()` -- these are SQLite data migration functions that already ran. For Postgres, it's a fresh database (per user's data policy: "existing dev data can be deleted without issues").

8. **Remove file system operations:** Delete `mkdirSync(dirname(dbPath))` -- no file-based DB.

Keep encrypted columns as TEXT type in the CREATE TABLE SQL (authContext, credentials, memory text/entities/claims/metadata).
