  <action>
Add `autoMerge()` method to `ContactsService` that implements three safety tiers:

**Algorithm:**

1. Load ALL contacts with their identifiers in two bulk queries (contacts + contactIdentifiers). Build a Map of contactId -> identifiers for O(1) lookup.

2. Group contacts by `displayName.toLowerCase().trim()`. Skip groups with < 2 members.

3. For each name group, apply rules IN ORDER:

**Tier 1 — Non-person exact name match (HIGH safety):**
- Filter group to contacts where entityType is in: organization, product, location, event, topic, pet, device, other
- If 2+ non-person contacts in the group, pick the one with the most identifiers as target, merge all others into it
- Track: `byRule.nonPerson++` per merge

**Tier 2 — Sparse-to-rich exact name match (HIGH safety):**
- For remaining contacts in the group (after Tier 1 merges), identify "sparse" contacts: those with ONLY 'name' type identifiers (check the identifiers map)
- Identify "rich" contacts: those with at least one non-name identifier (email, phone, slack_id, etc.)
- For each sparse contact, if there is exactly ONE rich contact with the same name, merge the sparse into the rich
- If multiple rich contacts match, skip (ambiguous — leave for manual review)
- Track: `byRule.sparseToRich++` per merge

**Tier 3 — Person-to-person (SKIP — leave for manual review):**
- Do nothing. These remain in getSuggestions().

**Important implementation details:**
- After each merge, remove the source contact from the group array (it no longer exists)
- Use the EXISTING `this.mergeContacts(targetId, sourceId)` — do NOT reimplement merge logic
- If mergeContacts throws (concurrent merge), catch and continue (best-effort)
- Re-query contacts at the START (not reuse stale data) to handle prior runs

**Return shape:**
```typescript
{
  merged: number,
  byRule: { nonPerson: number, sparseToRich: number },
  details: Array<{ targetId: string, sourceId: string, targetName: string, rule: string }>
}
```

**Also add `POST /people/auto-merge` endpoint** in the controller that calls `autoMerge()` and returns the result.

**Also modify `getSuggestions()`** to call `this.autoMerge()` at the very beginning (before loading contacts for suggestion generation). This ensures the user sees a clean queue. The auto-merge is idempotent so calling it multiple times is safe — it will return `merged: 0` if nothing to merge.

to load ALL contacts (no entityType filter). The auto-merge handles non-person dedup, and any remaining non-person suggestions are valid candidates for manual review.
