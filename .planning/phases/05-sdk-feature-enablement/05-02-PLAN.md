1. Add an `identifyUser` function to `apps/web/src/lib/posthog.ts` (which was updated by plan 05-01 with SDK feature config):

```typescript
export function identifyUser(
  userId: string,
  properties?: Record<string, unknown>,
): void {
  if (!apiKey) return;
  posthog.identify(userId, properties);
}
```

The `userId` should be the stable user identifier. For this single-user system, we use the email from `/api/me` response (`identity.email`), falling back to `identity.contactId` or `'botmem-user'` as a last resort. This ensures sessions are linked across page reloads.

2. In `apps/web/src/App.tsx`, add a `PostHogIdentifier` component that:
   - Runs once on mount (inside `AuthGuard` subtree so it only fires for authenticated users)
   - Fetches `GET /api/me`
   - Calls `identifyUser()` with `identity.email || identity.contactId || 'botmem-user'`
   - Sets person properties: `connectors_count` (from `accounts.length`), `memories_count` (from `stats.totalMemories`)

```typescript
import { identifyUser } from './lib/posthog';

function PostHogIdentifier() {
  useEffect(() => {
    fetch('/api/me')
      .then((r) => r.json())
      .then((data) => {
        const userId = data.identity?.email || data.identity?.contactId || 'botmem-user';
        identifyUser(userId, {
          connectors_count: data.accounts?.length ?? 0,
          memories_count: data.stats?.totalMemories ?? 0,
          name: data.identity?.name ?? undefined,
          email: data.identity?.email ?? undefined,
        });
      })
      .catch(() => {
        // Silently fail — analytics should never block the app
      });
  }, []);
  return null;
}
```

Place `<PostHogIdentifier />` inside the authenticated `<Route>` layout, as a sibling to `<Shell />` wrapped in a fragment:

```tsx
<Route
  path="/"
  element={
    <AuthGuard requireOnboarded>
      <>
        <PostHogIdentifier />
        <Shell />
      </>
    </AuthGuard>
  }
>
```

This ensures identify is called once per authenticated session, after the auth guard confirms the user is logged in.
