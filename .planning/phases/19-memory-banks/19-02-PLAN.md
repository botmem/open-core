  <action>
Create `apps/api/scripts/migrate-banks.ts` following the Phase 25 migration pattern (`main().catch()` for tsx CJS compatibility).

The script should:

1. **Connect to Postgres** using `pg` Pool with `process.env.DATABASE_URL`. Fail with clear message if not set.

2. **Find the first user**: `SELECT id FROM users LIMIT 1`. If no user exists, log "No users found, nothing to migrate" and exit cleanly.

3. **Ensure default bank**: `SELECT id FROM memory_banks WHERE user_id = $userId AND is_default = true LIMIT 1`. If not found, `INSERT INTO memory_banks (id, user_id, name, is_default, created_at, updated_at) VALUES (gen_random_uuid()::text, $userId, 'Default', true, NOW(), NOW()) RETURNING id`.

4. **Migrate accounts**: `UPDATE accounts SET user_id = $userId WHERE user_id IS NULL`. Log count.

5. **Migrate contacts**: `UPDATE contacts SET user_id = $userId WHERE user_id IS NULL`. Log count.

6. **Migrate memories**: `UPDATE memories SET memory_bank_id = $bankId WHERE memory_bank_id IS NULL`. Log count.

7. **Migrate Qdrant vectors**: Use the Qdrant REST API directly (no NestJS DI in scripts). `PUT /collections/memories/points/payload` with body `{ payload: { memory_bank_id: bankId }, filter: { must: [{ is_null: { key: "memory_bank_id" } }] } }`. Use `fetch()` with `process.env.QDRANT_URL || 'http://localhost:6333'`. Log result.

8. **Close pool** and log summary.

Make the script idempotent -- running it multiple times is safe (WHERE clauses only target NULL values).

Run command: `npx tsx apps/api/scripts/migrate-banks.ts`
</action>
