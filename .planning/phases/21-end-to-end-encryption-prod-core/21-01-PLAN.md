       - Add `updateEncryptionSalt(userId, newSalt)` method
       - Add `incrementKeyVersion(userId)` method that returns the new version

    2. Update `user-auth.service.ts`:
       - Import UserKeyService, inject in constructor
       - In `register()`: generate 16-byte salt via `randomBytes(16)`, pass to createUser, then call `userKeyService.deriveAndStore(user.id, password, salt)`
       - In `login()`: after bcrypt verify succeeds, fetch user's encryptionSalt from DB, derive and store key via `userKeyService.deriveAndStore(user.id, password, Buffer.from(salt, 'base64'))`
       - Handle legacy users with no encryptionSalt: if encryptionSalt is null on login, generate one, save it to DB, then derive key (this handles users created before E2EE)
       - Add `async changePassword(userId: string, oldPassword: string, newPassword: string)`:
         a. Fetch user, verify oldPassword with bcrypt
         b. Derive old encryption key: argon2id(oldPassword, existingSalt)
         c. Generate new salt (16 bytes), derive new encryption key: argon2id(newPassword, newSalt)
         d. Hash new password with bcrypt
         e. Update user: passwordHash, encryptionSalt (new), increment keyVersion
         f. Update UserKeyService with new key
         g. Enqueue re-encryption BullMQ job (to 'reencrypt' queue): { userId, oldKey: oldKey.toString('base64'), newKey: newKey.toString('base64'), newKeyVersion }
         h. Revoke all refresh tokens for user (force re-login with new password)
         i. Return immediately (re-encryption is async)

       - NOTE: Do NOT mark as @Public -- requires JWT auth
