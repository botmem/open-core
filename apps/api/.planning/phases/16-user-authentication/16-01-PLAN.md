<objective>
Build the backend user authentication system: database schema for users and refresh tokens, NestJS user-auth module with register/login/refresh/logout endpoints, JWT strategy, and guard infrastructure.

Purpose: This is the foundation for all v2.0 security features. Without user identity, nothing else (API keys, banks, encryption, RLS) can function. The existing auth module handles connector OAuth only -- this creates a separate user-auth module.

Output: Working backend auth endpoints that can register users, authenticate with JWT, handle refresh token rotation with family tracking, and provide guard infrastructure for protecting endpoints.
</objective>

  <behavior>
    - register: creates user with bcrypt(12) hash, returns { accessToken, user }, sets refresh cookie
    - register: rejects duplicate email with 409
    - register: rejects password < 8 chars with 400
    - login: valid credentials return { accessToken, user } + refresh cookie
    - login: invalid email or password returns 401 "Invalid credentials" (no email enumeration)
    - login: always runs bcrypt.compare even when user not found (timing attack prevention)
    - refresh: valid cookie returns new { accessToken } + rotated refresh cookie
    - refresh: revoked token triggers family revocation, returns 401
    - refresh: expired token returns 401
    - logout: clears refresh cookie, revokes token
    - me: GET /api/user-auth/me with valid Bearer returns user object
    - JwtStrategy extracts Bearer token, validates with jwtAccessSecret, returns { id, email }
    - JwtAuthGuard checks @Public() metadata, skips auth for public routes
  </behavior>
  <action>
    1. Create `users.service.ts`:
       - Injectable service with DbService injected
       - `createUser(email, passwordHash, name)` -- inserts into users table, returns user record
       - `findByEmail(email)` -- returns user or null
       - `findById(id)` -- returns user or null
       - `saveRefreshToken(userId, tokenHash, family, expiresAt)` -- inserts into refreshTokens
       - `findRefreshToken(tokenHash)` -- returns token row or null
       - `revokeRefreshToken(id)` -- sets revokedAt to now
       - `revokeTokenFamily(family)` -- sets revokedAt on all tokens in family
       - `revokeAllUserTokens(userId)` -- revokes all user's refresh tokens

    2. Create `user-auth.service.ts`:
       - Injectable with JwtService, UsersService, ConfigService injected
       - `register(email, password, name)`:
         - Validate password >= 8 chars, throw BadRequestException
         - Hash with bcrypt.hash(password, 12)
         - Create user via UsersService (catch unique constraint -> ConflictException)
         - Call generateTokenPair, return { accessToken, refreshToken, user }
       - `login(email, password)`:
         - Find user by email
         - Always run bcrypt.compare (use dummy hash if user not found per RESEARCH.md timing attack prevention)
         - If invalid, throw UnauthorizedException('Invalid credentials')
         - Call generateTokenPair, return { accessToken, refreshToken, user }
       - `refresh(oldToken)`:
         - Verify JWT with jwtRefreshSecret
         - Hash token with SHA-256, look up in DB
         - If not found -> UnauthorizedException
         - If revokedAt set -> replay attack: revoke entire family, throw UnauthorizedException
         - If expired -> UnauthorizedException
         - Revoke old token, generate new pair with same family
       - `logout(tokenHash)` -- revoke token
       - `generateTokenPair(userId, email, family?)`:
         - Sign access token (15min, HS256, payload: { sub: userId, email })
         - Generate refresh token: crypto.randomBytes(32).toString('hex'), sign as JWT with jwtRefreshSecret (7d)
         - Hash refresh token value with SHA-256, store in DB with family UUID
         - Return { accessToken, refreshToken }

    3. Create `jwt.strategy.ts`:
       - PassportStrategy(Strategy, 'jwt')
       - Extract from auth header as Bearer token
       - secretOrKey from config.jwtAccessSecret
       - algorithms: ['HS256']
       - validate returns { id: payload.sub, email: payload.email }

    4. Create `jwt-auth.guard.ts`:
       - Extends AuthGuard('jwt')
       - canActivate checks IS_PUBLIC_KEY via Reflector on handler and class
       - If public, return true; else call super.canActivate

    5. Create `user-auth.controller.ts` with prefix 'user-auth':
       - POST /register: body { email, password, name }, calls service.register, sets refresh cookie (httpOnly, secure in prod, sameSite strict, path /api/user-auth, maxAge 7d), returns { accessToken, user }
       - POST /login: body { email, password }, same cookie pattern, returns { accessToken, user }
       - POST /refresh: reads req.cookies['refresh_token'], calls service.refresh, sets new cookie, returns { accessToken }
       - POST /logout: reads cookie, revokes token, clears cookie, returns { ok: true }
       - GET /me: @UseGuards(JwtAuthGuard), @CurrentUser() user, returns user from DB

    6. Create `user-auth.module.ts`:
       - Import PassportModule, JwtModule.registerAsync (using ConfigService for secret, 15m expiry)
       - Providers: UserAuthService, UsersService, JwtStrategy, JwtAuthGuard
       - Controllers: UserAuthController
       - Export JwtAuthGuard, JwtStrategy (for Phase 17 global guard)

    7. Add UserAuthModule to AppModule imports in app.module.ts

    8. Create test file `__tests__/user-auth.service.test.ts`:
       - Test register creates user with hashed password
       - Test login with valid credentials returns tokens
       - Test login with wrong password throws UnauthorizedException
       - Test login with non-existent email throws UnauthorizedException
       - Test refresh rotates token and returns new access token
       - Test refresh with revoked token revokes entire family
       - Test register rejects short passwords
